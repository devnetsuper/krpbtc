Introduction

KRP BTC is a framework for verifying cryptocurrency transactions. It provides a guide for financial professionals that ensures:

  • Best audit practices set by Canadian regulators.
  • Efficient ways of verifying cryptocurrency transactions on-chain.

The main value props of KRP BTC are:

  • Internal Libraries: most cryptocurrency transactions settle on open, permissionless blockchains that allow us to develop scripts that extract raw data.
  • Data Classification: the raw data extracted is then organized in an accountant-friendly manner integrated with Excel.

Work Specialization

Here are the technical controls set in place to meet the industry standards and regulations for the auditing of cryptocurrencies.

Verification of transactions

  • Ensure all transactions have been settled on the blockchain.
  • Match data provided by client, with a focus on date of transaction, transaction hash, amount transacted, asset type and destination address.
  • Determine the status of the transaction as verified or invalid.

Transaction dashboard

  • Organize data clumps in an accountant-friendly manner.
  • Label transactions as incoming or outgoing.

Daily price and historical data

  • Get the closing price of major assets at any time frame.

Transaction rollback

  • Determine the balance of any wallet address at year-end (or custom date).
  • Perform any tasks related to ownership ceremonies, such as ownership ceremony rollbacks.

Unrealized gains or losses

  • Provide the closest P/L estimate per transaction for every major asset, and inevitably get the closest estimate of wallet address P/L.

Revenue or testing models

  • Test revenue correctness by creating scripted calculators.
  • Test mining revenue by creating mining output models dependant on network hash rate, hardware and block rewards.

Internal Bitcoin Node and Blockchair Infrastructure

  • Operate and monitor a full bitcoin node to minimize the reliance of trust on third parties.
  • For every other network, we rely on Blockchair's infrastructure and purchase a monthly API key that grants us 5, 000 requests daily.

Hardware

How to Prepare Client Files

The client will provide us with a .csv file containing the raw data from their wallet addresses. To verify the transactions on the blockchain, we can focus on the following data points:

Transaction Hash Server Time Transaction Value Asset Destination Address

We can also request the client's addresses and retrieve these data internally.

Mining Operations and Activities

The strategy to test revenue is almost identical for all proof-of-work networks. There are three components that will determine the profitability of operations:

  • Hardware capacity
  • Client's network hash rate share
  • Operational expenses (electricty, mining fees)

Hardware capacity

Since mining is a highly monopolized field, most likely the mining equipment is being made by the same companies. All we are concerned with is determining 3 specific characteristics of any mining equipment:

  • Hash rate capacity
  • Power consumption
  • Mining algorithm

The hardware specifications are easy to gather. On your web browser type “equipment model in question + specs”.

Client's network hash rate share

The formula to calculate our client’s percentage of the total network hash rate that their miner represents is the following:

Client's hash rate share = equipment hash rate/ daily network hash rate

Now, we have to determine the total daily block reward. So for example, a Bitcoin block is mined every 10 minutes or so:

Daily block reward = Mining reward * blocks per day

Finally, we have to determine the amounts of coins we will get according to our mining capacity:

Client’s daily coins = total daily block reward * client’s hash rate share

Mining algorithm

The mining algorithm will determine the asset being mined. For example, if our client mines using NVIDIA GPUs, we automatically know they are not mining for Bitcoin. Maybe back in the early years, when Bitcoin mining wasn’t as competitive, however, now it would be virtually impossible to compete against ASIC miners.

Introduction to Open Bitcoin for Excel

Open Bitcoin for Excel (“openbtc” for short) is an open-source web application that provides detailed wallet information about prices, transactions, and blockchain confirmations more directly into Excel. It was born from a lack of existing libraries.

Project Description and Capabilities

  • Address checker: verify if an address is real and settles on-chain.
  • Transaction verification: upload an .xlsx file with bitcoin transactions and receive an output file with their respective block explorer URL.
  • Transaction hash search: enter a transaction hash and receive a block explorer URL if the hash is found on-chain.
  • Retrieve historical transactions: enter a bitcoin address and select the custom date range for transaction search.
  • Calculate estimated unrealized gains or losses
  • Historical price data: retrieve the closing price of bitcoin at a custom date range from major exchanges.

Introduction to terminalWallet

terminalWallet is a command-based python program that retrieves historical wallet information for addresses found in major networks such as Bitcoin, Ethereum, Avalanche, Polygon, and Binance Smart Chain. The program retrieves the data from Blockchair and Block Daemon.
